# Защита изображений стафа

## Реализованная защита

Для аватарок стафа реализована многоуровневая защита от копирования и скачивания:

### 1. HTML атрибуты
- `draggable={false}` - запрет перетаскивания изображения
- `onContextMenu={(e) => e.preventDefault()}` - блокировка правой кнопки мыши

### 2. CSS защита
```css
img[alt*="avatar"] {
  user-select: none;           /* Запрет выделения */
  pointer-events: none;        /* Отключение событий мыши */
  -webkit-user-drag: none;     /* Запрет перетаскивания в WebKit */
  -webkit-touch-callout: none; /* Запрет долгого нажатия на iOS */
}
```

### 3. Прозрачный защитный слой
Поверх каждой аватарки добавлен невидимый `div`, который:
- Перехватывает все клики мыши
- Блокирует контекстное меню
- Предотвращает перетаскивание

## Что защищено

✅ Правая кнопка мыши (контекстное меню)
✅ Перетаскивание изображения
✅ Выделение изображения
✅ Копирование через Ctrl+C
✅ Долгое нажатие на мобильных устройствах
✅ Сохранение через браузерные расширения (частично)

## Ограничения

⚠️ **Важно понимать:**
Эта защита затрудняет, но не делает невозможным копирование изображений. Опытный пользователь всё равно может:
- Открыть инструменты разработчика (F12) и найти URL изображения
- Сделать скриншот экрана
- Использовать специализированные инструменты для захвата изображений

## Дополнительные меры (опционально)

Если нужна более серьёзная защита, можно добавить:

1. **Водяные знаки** - добавить полупрозрачный текст поверх изображения
2. **Серверная обработка** - генерировать изображения с водяными знаками на сервере
3. **Canvas рендеринг** - отрисовывать изображения через Canvas вместо `<img>`
4. **Проверка Referer** - ограничить доступ к изображениям только с вашего домена

## Тестирование

Проверьте защиту:
1. Попробуйте кликнуть правой кнопкой на аватарку → должно быть заблокировано
2. Попробуйте перетащить аватарку → не должно работать
3. Попробуйте выделить аватарку → не должно выделяться
4. Попробуйте скопировать через Ctrl+C → не должно копироваться
